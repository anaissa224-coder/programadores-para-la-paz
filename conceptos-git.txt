Repositorio: Es el espacio digital, o "contenedor", donde se almacena todo el proyecto, incluyendo archivos, carpetas y, lo más importante, el historial completo de cambios. Existen dos tipos: el local, que reside en tu propia computadora, y el remoto, alojado en plataformas como GitHub o GitLab para facilitar la colaboración. Funciona como una base de datos que permite reconstruir cualquier versión anterior del código en cualquier momento.
Commit: Se define como una "instantánea" o fotografía del estado actual de tus archivos en un momento específico. Al realizar un commit, guardas de forma permanente los cambios que has seleccionado en el historial del repositorio, acompañados de un mensaje descriptivo que explica qué se hizo. Es la unidad básica de progreso en Git, permitiéndote tener un registro cronológico exacto de la evolución del software.
Push: Es el comando que se utiliza para cargar y transferir todos los commits realizados en el repositorio local hacia un repositorio remoto en la nube. Su función principal es actualizar el servidor central con tus últimos cambios para que otros miembros del equipo puedan verlos y descargarlos. Es el paso esencial para sincronizar el trabajo individual con el proyecto colectivo y asegurar que el código esté respaldado fuera de tu máquina.3
